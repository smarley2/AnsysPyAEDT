# Contributing

Thank you for contributing to PyAEDT Inductor Designer.

## Before starting

1. Read `AGENTS.md`.
2. Read the approved design specification and active implementation plan.
3. Open or claim a GitHub Issue with explicit acceptance criteria.
4. Create a dedicated branch and working tree.

## Development requirements

- Use English for all repository content.
- Add tests before implementation.
- Preserve the dependency boundaries documented in `docs/architecture/README.md`.
- Cite every added catalog or material value with a redistributable source.
- Never commit proprietary datasheets or generated AEDT artifacts.
- Use conventional commit messages.
- Include verification commands and results in the pull request.

## Review requirements

A change is ready for review when its acceptance criteria pass, public interfaces are documented, migrations are included for schema changes, data provenance is complete, and no unrelated files are modified.
